# Audience & Personas — v0.2 用户边界

本文把 v0.2 的用户范围写死，避免产品方向在“自用研究工具、开源项目、投资仪表盘、媒体展示页”之间摇摆。

## 决策

v0.2 只服务两类用户：

```text
A. 自用研究员：项目维护者本人，需要更快做供应链研究。
B. 复现型外部研究员：能读公开披露，愿意在本地跑开源工具，目标是复现一份证据链。
```

v0.2 不服务：

```text
C. 行业研究员 / 投资人：需要漂亮 dashboard 或投资结论。
D. 记者 / 监管观察者：需要新闻 hook 或公开展示材料。
E. 普通散户：需要低门槛解读或交易建议。
```

## 产品决策尺子

任何 v0.2 功能都按下面问题判断：

```text
让 A 工作更快？做。
让 B 第一次跑通更顺利？做。
只是让 C 看起来更专业？暂缓。
只是给 D 一个展示故事？暂缓。
需要用户系统、多租户、权限、商业化包装？不做。
```

## v0.2 必须优化的体验

### A：自用研究员

需要：

- 一条命令看到某家公司最新供应链边。
- 快速知道最近哪些源变了、哪些边新增或失效。
- 能审计 evidence、review candidate、unknown map。
- 能把输出嵌入自己的研究笔记。

不需要：

- 登录。
- 团队协作。
- 漂亮动画。
- 投资建议。

### B：复现型外部研究员

需要：

- 从 README 到第一份 NVIDIA 研究输出的路径清晰。
- 本地失败时错误可解释。
- 证据、源码、数据源边界都能查。
- 能看到静态研究工作台，理解项目长什么样。

不需要：

- 托管 SaaS。
- 复杂部署。
- 自定义看板。
- 自动交易或选股。

## 功能取舍

| 功能                               | v0.2 决策 | 理由                                         |
| ---------------------------------- | --------- | -------------------------------------------- |
| ComponentCard / `cli component`    | 做        | A/B 都需要按组件研究链路                     |
| `cli changes`                      | 做        | 研究员每天最关心“有什么变了”                 |
| `cli search`                       | 做        | 本地研究需要快速找实体、边、证据             |
| `apps/research-preview`            | 做        | B 第一次接触必须看到真实工作台形态           |
| Apple Supplier List facility edges | 做        | 最快产生高等级设施边，验证 review/apply 价值 |
| SEC 10-Q / 8-K                     | 做        | 监控变化必须有更高频监管文件                 |
| DART-KR                            | P1        | 对 Samsung / SK Hynix 的交叉验证 ROI 高      |
| LLM 真实抽取                       | v0.2 搁置 | 规则边界未稳定前，LLM 会增加不可复现拐点     |
| 投资 dashboard                     | 不做      | 本仓库是事实层，不输出投资建议               |
| 用户登录 / 多租户                  | 不做      | A/B 本地使用，不需要团队产品复杂度           |

## 反模式

- 为了让页面看起来完整，展示无证据的二级/三级边。
- 为了吸引投资人，提前做漂亮 dashboard。
- 为了“智能”，在 v0.2 启用 LLM 自动抽取。
- 为了开源热闹，把 onboarding 做成营销页而不是可复现研究路径。
