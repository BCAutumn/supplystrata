# Ethics — 伦理边界

写在工程上：**很多事情技术上能做不代表应该做**。

## 核心原则

1. **不做不诚实的输出**。系统不要假装比实际更确定。
2. **不输出投资建议**。本系统目标是事实图谱，不是 alpha 信号。第三方拿来做投资是他们的责任。
3. **不让 LLM 编造关系**。LLM 抽取必须 cite 原文；任何无 cite 的 LLM 关系一律拒绝。
4. **不监控个人**。本系统是公司层 / 组件层 / 设施层，不下沉到个人。
5. **不替任何机构背书**。不通过本系统的输出做"X 公司有问题 / Y 公司是黑名单"等价值判断。
6. **不规避 ToS / 法律边界以求"更全的数据"**。

## 关于"投资推断系统"的边界

如果未来有一个独立项目消费本系统的输出做投资决策：

- 那是另一个项目的事
- 它必须自己负责其法律合规与策略风险
- 本系统不直接输出投资建议
- 本系统在 README / 输出中明确说明"非投资工具"

## 关于"自动化抓取"

- 公开数据 + 官方 API + 限速 + UA + 注明 → 没问题
- 公开数据 + 在 robots.txt 允许的 HTML → 通常没问题
- 反爬严格但允许人工查询的网站 → **不**做自动化
- 明确禁止自动化的网站 → **不**做自动化
- 通过技术手段绕过反爬（例如轮换 IP、头部欺骗）→ **不**做

## 关于"LLM 替代研究"

LLM 在本系统里只做：

- 候选关系抽取
- 实体消歧辅助
- 文本片段定位

LLM 不做：

- 凭空给出"行业知识"
- 推断没有原文支撑的关系
- 替代研究员的判断

## 关于公开发布

- 仓库代码可以公开
- 抽取规则可以公开
- 图谱片段（脱敏后）可以公开作为 demo
- **不公开**原始 PDF / HTML 字节
- **不公开**未脱敏的 LLM prompts/responses（避免泄漏 internal heuristics）

## 关于"对手"行为

可能有人想用本系统：

- 找出某公司 / 某国家的供应链弱点 → 不在意图，公开数据本就如此
- 用作竞争情报 → 我们提供事实图谱
- 用作攻击规划 → 不接受。任何明显恶意用途的请求会被拒绝

## 不接受的请求

下列请求即使技术上可行，本系统拒绝实现：

- "帮我自动监控某个高管的行程 / 邮件"
- "找出某公司的弱点供应商以便攻击"
- "替我做股票自动交易"
- "为某政治目的输出名单"

## 个人责任

- 维护者对自己的提交负责
- LLM 输出错了不能甩锅给"模型"——是系统设计要保证 evidence 链的完整
- 数据质量责任（不变量校验、抽样审计）也在维护者

## 内部 reflection

每月（或每个 phase 结束）做一次伦理 reflection：

- 是否有为了"看起来更完整"而牺牲诚实度？
- 是否有 evidence_level 被偷偷上调？
- 是否有 unknown_map 被刻意写得"很少"？
- 是否有 LLM 的输出被当作事实使用而无 cite 校验？

发现问题 → 在 `docs/07-operations/audit-log.md` 记录 + 修正。
