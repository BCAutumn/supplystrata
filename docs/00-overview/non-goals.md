# Non-Goals — 第一版**不做**什么

明确写下不做什么，比写做什么更重要。第一阶段任何"顺手做一下"都会拖垮主线。

## 一、产品层面不做

| 不做                  | 原因                                              |
| ------------------- | ----------------------------------------------- |
| 自动选股 / 自动交易信号       | 第一阶段目标是事实图谱，不是 alpha 系统                         |
| 自动回测                | 没有可信的关系图，回测毫无意义                                 |
| 投资建议 / 看多看空         | 法律灰色 + 与系统定位冲突                                  |
| 漂亮的 Web 前端          | MVP 用 CLI + Markdown / JSON 输出即可                |
| 移动端 / 推送 / 邮件订阅     | 偏离主线                                            |
| 用户系统 / 多租户 / SaaS 化 | 个人研究工具优先，留接口但不实现                                |
| 实时监控 / 秒级延迟         | 数据源本身大多是日级、月级、季度级                               |
| 自然语言对话式问答 (Chat UI) | 首阶段不接，避免幻觉成本太高；查询全部走结构化 CLI / API              |

说明：不做"投资建议"不等于不服务金融场景。SupplyStrata 的事实图谱可以作为投研、风控、产业链监控、组合研究系统的输入，但仓库本身只输出可审计事实、证据、未知项和变化事件，不输出买卖结论。

## 二、数据层面不做

| 不做                          | 原因                                              |
| --------------------------- | ----------------------------------------------- |
| 商业付费数据库（Bloomberg / IHS 等）  | 与"免费公开优先"冲突                                     |
| TrendForce / S&P 完整库        | 仅引用其公开新闻稿作为证据，不接付费 API                          |
| 全量全球 AIS                    | 数据量过大、收益不确定，先做 NOAA 美国水域                        |
| 卫星图像分析                      | 工程量巨大，留到 Phase 5+                               |
| 公司内部 ERP / 订单数据             | 拿不到，且越界                                         |
| 可能违反 ToS 的爬虫（ImportYeti 自动化等） | 法律灰色，详见 [legal-tos.md](../09-risks-compliance/legal-tos.md) |
| 社交媒体情绪监测                    | 信号噪声太大，且容易被操纵                                   |

## 三、技术层面不做

| 不做                       | 原因                                              |
| ------------------------ | ----------------------------------------------- |
| 微服务拆分                    | 单进程 + 队列就够用，过早拆分微服务是反模式                         |
| Kubernetes / 复杂基建        | docker-compose 就能跑                              |
| 自建 LLM / 自训 NER 模型       | 成本与收益不成比例，调用商业 API 即可                           |
| 自建图谱数据库                  | Neo4j Community Edition 完全够用                    |
| 自研全文检索                   | Postgres full-text 先够用；OpenSearch 等 Phase 3 之后再说 |
| GraphQL 层                | REST + JSON Schema 简单可靠                         |
| 多语言 i18n                 | 首期中英双语手写即可                                      |

## 四、抽取/推断层面不做

| 不做                          | 原因                                              |
| --------------------------- | ----------------------------------------------- |
| LLM 凭空补全没有原文支撑的关系           | 严禁。所有 LLM 输出必须 cite 原文片段                        |
| 把 Level 1-3 的推断当结论          | 必须始终标 `is_inferred = true`                      |
| 自动 promote 边的 evidence_level | 升级必须经过显式审核流程，不能自动触发                             |
| 跨语言关系抽取的"通才"流水线             | 第一版只针对英文 + 简单中文披露，韩文 / 日文 / 德文 IR 文件需要专门 adapter |

## 五、研究范围层面（首阶段）不做

| 不做           | 何时再考虑                  |
| ------------ | ---------------------- |
| 全行业覆盖        | 首版只做 AI 算力 / 半导体 / 内存链 |
| 新能源 / 锂电池链   | Phase 3+               |
| 机器人 / 人形机器人链 | Phase 4+               |
| 国防军工         | 法律风险大，暂缓               |
| 农产品 / 大宗商品   | 与主线协同性低                |

新能源链虽然在原始需求里被提到，但放进首阶段会摊薄精力。先把一条链做透。

## 六、运营层面不做

- 不做用户增长
- 不做营销
- 不做对外公开（除非 Phase 4 之后明确决议）
- 不做付费订阅

---

## "什么时候可以重新评估这些 non-goals"

每完成一个 Phase（见 [roadmap.md](../06-development/roadmap.md)）后，重新审视本文件。如果要改某一条，必须开 ADR。
