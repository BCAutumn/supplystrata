# Vision — 项目愿景与定位

## 一句话定位

> **用免费/公开数据，把全球关键产业链整理成一张可追溯、带证据等级、带时间戳的图谱。**

## 它要回答的问题

```
某公司依赖哪些上游？
某产品依赖哪些组件？
某组件由哪些公司、地区、工厂提供？
这些关系是直接披露、第三方披露、贸易数据推断、还是新闻推断？
证据在哪里？
证据是什么时候采集的？
最近有没有变化？
哪些环节我们知道？哪些环节其实不知道？
```

## 它**不**直接回答的问题

```
明天该买什么股票？
NVIDIA 下一季度业绩会不会超预期？
某只股票几个月后涨多少？
某公司明年的具体订单数量？
某批 GPU 的具体客户分配？
```

这些问题不是没有价值，而是**它们不应该是 MVP 的目标**。如果一开始就把目标设成"投资 alpha"，会强烈诱导系统输出"看起来确定但其实没有证据"的结论。本系统刻意把目标限定在**事实图谱**层面，把任何投资推断推到第二阶段。

## 三阶段优先级

```
第一目标：把供应链链条梳理清楚（事实层）
第二目标：让每条边都有证据 / 等级 / 时间戳 / 来源（证据层）
第三目标：再讨论投资推断和回测（推断层）
```

第一、第二目标完成度不达标之前，**禁止**在系统输出里出现"建议买入 / 建议看多 / 提前布局"等措辞。

## 核心原则

### 原则 1：证据等级不是装饰

每条供应链关系必须挂 `evidence_level` (1-5)。低于 4 的关系一律标 `is_inferred = true`。详见 [03-data-model/evidence-model.md](../03-data-model/evidence-model.md)。

### 原则 2：未知比已知更重要

任何研究输出都必须附带"未知地图"（unknown map）：明确告诉用户**这个问题里我们不知道什么**。这比给出一个伪精确答案更有价值。详见 [08-research-workflow/unknown-map.md](../08-research-workflow/unknown-map.md)。

### 原则 3：免费公开优先

P0 数据源必须是真正免费、真正公开、真正合法可用的。商业数据库（IHS、S&P Global、Bloomberg、PitchBook、TrendForce 完整版）一律不进 P0。

### 原则 4：实体消歧先于关系抽取

`Foxconn / Hon Hai / 鸿海 / 富士康 / FIH Mobile / Hongfujin Precision` 这类问题如果没解决，后续所有图谱都是错的。详见 [05-modules/entity-resolution.md](../05-modules/entity-resolution.md)。

### 原则 5：LLM 是抽取助手，不是事实生成器

LLM 只用于：从原文抽取候选关系、给出证据句、标注不确定性。它**不**用于：凭空补全公司间关系、回答"你觉得谁是谁的供应商"这种没有原文支撑的问题。

### 原则 6：可追溯到原文

任何边、任何卡片、任何报告里的每一句结论，都能 1-click 跳到原始文档（PDF / HTML / API response）的对应位置。这是图谱可信度的底线。

## 系统的"用户"是谁

短期内（Phase 0-2）只有一类用户：

- **作者本人 / 同等水平的研究者**：能读懂原始文件、能判断证据强弱、愿意手动审核 LLM 抽取结果。

长期（Phase 3+）才考虑：

- 投研分析师
- 行业研究员
- 学术研究

**不**面向：

- 散户股民
- 自动交易机器人
- 不读证据只看结论的用户

这影响很多产品决策：MVP 阶段没必要做漂亮 UI，没必要做推送，没必要做手机端。

## 成功标准

第一阶段成功的判断不是 GMV、不是 DAU，而是：

1. 对一个具体公司（如 NVIDIA），能否生成一份所有边都有 ≥ Level 4 证据的一级上游图？
2. 对一个具体组件（如 HBM），能否给出有时间戳、有来源、有不确定性标注的供需图？
3. 同一个实体在不同来源中（SEC、IR、Comtrade、新闻）能否被正确合并为同一个节点？
4. 当数据缺失时，系统是否会"老老实实承认不知道"，而不是 LLM 编一个？

如果上面四条不能稳定做到，就不应该往"投资推断"层迈步。
