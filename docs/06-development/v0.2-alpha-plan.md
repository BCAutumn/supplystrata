# v0.2 Alpha Plan — 从可运行切片到可复现研究工具

本文定义 v0.2-alpha 的产品和工程范围。它不是 Phase 2 的全部完成定义，而是一个可以发布、可以被外部研究员复现的中间版本。

## v0.2 定位

v0.2 的目标：

```text
让一个外部研究员在本地复现 NVIDIA 供应链研究，并看到：
1. 高等级事实边
2. evidence trace
3. changes / source health
4. component card
5. chain-first research workbench
```

v0.2 不追求：

```text
完整 25 节点覆盖
漂亮投资 dashboard
LLM 自动抽取
宏观/物流/矿产源大规模接入
```

## P0：不做不能 release

### 1. Apple Supplier List facility edges

当前代码状态：

```text
SupplierListReviewPayload 已经包含 facility_relation_hint = MANUFACTURES_AT
review apply 已经会从同一条 approved supplier-list row 写入 buyer BUYS_FROM supplier
review apply 同时会稳定创建 facility entity，并写入 supplier MANUFACTURES_AT facility
```

因此 v0.2 的剩余实现重点不是“再解析 PDF”，而是把这条 apply 路径批量跑起来，补齐 seed/alias，让足够多的 Apple supplier row 能通过 review/apply 变成真实 facility edges。

验收：

```text
[ ] 至少 50 条 `MANUFACTURES_AT` 或 facility-related Level 4 edge
[ ] 每条 edge 有 cite_text、locator、reviewer
[ ] 未 review 候选不入 graph
[x] review apply / batch apply 都落 ChangeRecord
[x] facility entity 有稳定 ID、country_or_region、source row provenance
```

### 2. SEC 10-Q / 8-K 接入

验收：

```text
[x] `sec-edgar` 能 plan/fetch/normalize 最新 10-Q
[x] `sec-edgar` 能 plan/fetch/normalize 最近若干 8-K
[x] source monitor 记录 DOCUMENT_NEW / DOCUMENT_CHANGED / DOCUMENT_UNCHANGED
[ ] 至少 1 条新 evidence 或 change 能从 10-Q/8-K 触发
```

实现边界：`sec-edgar` adapter 的 `formTypes` 支持 `10-K / 10-Q / 20-F / 8-K`，`limit` 可让 plan 一次吐出多份最近 8-K task；`sources check --source sec-edgar` 已接入 source monitor，会保存 normalized document、记录 DOCUMENT_NEW/UNCHANGED/CHANGED，并抽取 observation。`source_check_targets` 和 `sources run-due` 已经把“到期检查”从手工命令推进到可配置调度入口；pipeline shortcut 仍默认只跑 NVIDIA 10-K。10-Q / 8-K 进一步触发 relation extraction / edge changes 需要后续把 source check 输出接入规则抽取调度。

### 3. `cli changes`

验收：

```text
[x] `supplystrata changes --since <date>` 可读输出
[x] `apply` / `review apply` / `deprecate` / Apple supplier import 都写 ChangeRecord
[x] changes 输出能指向 edge_id / evidence_id / source_item_id
```

第一版 `changes` 已合并 `change_records` 与 `source_change_events`，支持 `--scope`、`--source`、`--type`、`--attention-only`、Markdown/JSON。`graph apply`、review approve/reject/apply/block、unknown add/resolve、edge deprecate、source failure/recovery 和 Apple supplier import/apply 都会写入 timeline。剩余缺口转向文档段落级 semantic diff。

### 4. ComponentCard + `cli component`

验收：

```text
[x] `supplystrata component memory --format markdown`
[x] 输出 known_suppliers / known_consumers / unknown_map
[x] 每条事实性 edge 挂 primary_evidence_id（如果 edge 已有 primary evidence）
[x] 默认隐藏 Level 1-3 inferred 内容
```

当前实现是通用组件卡片：按 `components` taxonomy 和 `edges.component_id/component` 聚合，不绑定 Apple / NVIDIA。CLI 另新增 `supplystrata chain <company> --depth N`，用于任意公司的一至多级 upstream chain 展开；在数据还只有一跳时会诚实显示一跳，后续 DART/EDINET/IR/供应商名单进入后自然扩展到二级、三级。

### 5. EntityResolver golden set

验收：

```text
[x] golden set >= 200
[x] CI 跑通
[x] 高歧义实体覆盖 Samsung / Foxconn / TSMC / 3M / Apple / Micron / SK
[x] false merge 零容忍；宁可 unknown
```

当前 golden set 先由 `seeds/entities.csv` 与 `seeds/aliases.csv` 派生，覆盖 active 实体的 canonical/display/entity_id/ticker/强别名，并额外加高风险歧义样本。下一步再沉淀人工维护 fixture，专门覆盖新接数据源里的误合并案例。

### 6. `apps/research-preview`

验收：

```text
[x] 替代新的静态 HTML 脚本扩张；`scripts/render-research-html.mjs` 仅保留临时报告用途
[x] 全量 TypeScript + Canvas
[x] 不引入 React
[ ] 支持公司切换
[x] Chain graph 点击边 -> Evidence drawer
[x] Source health 区块
[x] Changes timeline
[x] Unknown boundary
[x] 支持 `?report=/reports/<file>.json` 自动加载本地 workbench export
```

当前工作台消费 `@supplystrata/workbench-export` 的 truth-store JSON，不再把 NVIDIA preview report 当成长期前端契约。公司切换仍待做，因为它需要 `workbench export` 同时携带多个 root company 或在本地服务上补一个只读 manifest。

`claims / observations / chain_segments` 的中期骨架详见 [midterm-intelligence-network-plan.md](./midterm-intelligence-network-plan.md)。v0.2 可以先完成工作台 preview；中期骨架从 v0.2 之后接力，不为了提前做大范围宏观源而阻塞 release。
